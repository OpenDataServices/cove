{% trans "Invalid 'uri' found" %}

{% blocktrans %}This message means you have a problem, usually with a website address in your data.
Common mistakes include:{% endblocktrans %}

  • {% blocktrans %}Not starting the address with 'http://' e.g. example.com is not a 'uri' but http://example.com is.{% endblocktrans %}
  • {% blocktrans %}Having extra spaces at the start or end of the address - these should be removed{% endblocktrans %}
  • {% blocktrans %}Putting more than one address in the same field e.g. "http://example.com and http:example2.com" is not a 'uri'{% endblocktrans %}

{% trans "Date is not in the correct format" %}

{% blocktrans %}Dates should be in the formats defined by the 360Giving Standard.{% endblocktrans %}

{% blocktrans %}Most dates should be in the YYYY-MM-DD format, but the 360Giving Standard sets out the requirements for each field of the standard.{% endblocktrans %}

{% blocktrans %}Other common errors include{% endblocktrans %}

  • {% blocktrans %}providing non-existent dates e.g. 2016-02-31 (31st February){% endblocktrans %}
  • {% blocktrans %}non-date content in a date field (by accident){% endblocktrans %}

{% blocktrans %}Datetimes (used for Last Modified fields){% endblocktrans %}

{% blocktrans %}A datetime format is a very specific form of a date and time that is specified by the 360Giving Standard.{% endblocktrans %}

{% blocktrans %}A datetime should be in the form of YYYY-MM-DDThh:mm:ss±hh:mm where:{% endblocktrans %}

  • {% blocktrans %}YYYY represent a year e.g. 2016{% endblocktrans %}
  • {% blocktrans %}MM represents a 2 digit month of the year e.g. 01 for January{% endblocktrans %}
  • {% blocktrans %}DD represents a 2 digit day of the month e.g. 01 for the 1st{% endblocktrans %}
  • {% blocktrans %}T is simply the letter 'T'{% endblocktrans %}
  • {% blocktrans %}hh is a 2 digit representation of an hour e.g. 01 for 1AM, 13 for 1PM{% endblocktrans %}
  • {% blocktrans %}mm is a 2 digit representation of minutes in an hour e.g. 05 for 5 past{% endblocktrans %}
  • {% blocktrans %}ss is a 2 digit representation of seconds{% endblocktrans %}
  • {% blocktrans %}±hh:mm is a timezone offset from UTC. For most UK data producers this can simply be replaced with a 'Z'{% endblocktrans %}

{% blocktrans %}Example: 2016-05-15T12:00:00Z is 12 noon on the 15th May 2016 in UTC time{% endblocktrans %}
